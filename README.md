# LoL Esports Live Viewer

Десктопное приложение для просмотра live-статистики матчей LoL Esports. Подключается к официальному API lolesports.com и отображает данные в реальном времени.

## Возможности

- Расписание матчей — активные, предстоящие и завершённые
- Live-статистика карты: золото, киллы, башни, бароны, драконы, разница в голде
- Выбор конкретной карты матча вручную
- Окно KDA с результатами игроков
- Запись статистики киллов в локальный CSV
- Синхронизация CSV с Google Sheets (опционально)

## Использование

### Расписание

Нажми **Refresh schedule** — в левой панели появятся две вкладки:
- **Live + Upcoming** — текущие и предстоящие матчи
- **Finished** — завершённые матчи за последние ~2 недели

Кликни на любой матч — справа загрузится статистика.

### Статистика карты

Отображается в правой панели:

```
timestamp : 2026-03-15 14:32:10
gameState : in_game
gameTime  : 28:14

T1 (BLUE): K=14 G=52300 T=6 B=1 I=1 D=INF CLD MTN
GEN (RED) : K=8  G=44100 T=3 B=0 I=0 D=OCN
Gold diff : +8200 (T1)
```

### Управление

| Элемент | Описание |
|---|---|
| **Game** | Выбор карты матча (Auto = текущая активная) |
| **Flip sides** | Поменять RED и BLUE местами |
| **Anchor (-sec)** | Сдвиг временной метки запроса назад в секундах |
| **-10 / +10** | Быстрое изменение anchor |
| **Show KDA** | Открыть окно с K/D/A игроков |
| **Save JSON** | Сохранить все данные матча в JSON файл |
| **Write KDA** | Записать статистику открытого матча в CSV |
| **Write All KDA** | Записать статистику всех завершённых матчей из расписания |

### Окно KDA

Показывает киллы, смерти и ассисты каждого игрока по текущей карте. Обновляется вместе с основным polling. Масштаб шрифта — **Ctrl + колесо мыши**.

### Anchor

API отдаёт данные с задержкой относительно реального времени. Параметр Anchor сдвигает запрос назад — например `-30` означает "дай мне данные на 30 секунд назад от текущего момента". Если данные не обновляются — попробуй увеличить значение.

## Запись статистики (CSV)

Файл `kda_stats.csv` создаётся рядом с `main.py` автоматически при первой записи.

### Структура

| Колонка | Описание |
|---|---|
| `game_num` | Номер карты в матче |
| `team` | Код команды |
| `player` | Имя игрока (пустое = строка команды) |
| `champion` | Чемпион |
| `kills` | Киллы |
| `tournament` | Название лиги |
| `date` | Дата карты (UTC) |
| `match_id` | ID матча |
| `game_id` | ID карты |

Строка с пустым `player` — суммарные киллы команды за карту. Строки с именем — статистика каждого игрока.

### Защита от дублей

При повторной записи того же матча старые строки с совпадающими `game_id` автоматически удаляются и заменяются новыми.

## Google Sheets (опционально)

Позволяет нескольким пользователям вести общую таблицу статистики.

### Настройка

1. Открыть [Google Cloud Console](https://console.cloud.google.com) → создать проект
2. Включить **Google Sheets API** и **Google Drive API**
3. IAM → Service Accounts → создать → скачать JSON ключ → положить рядом с `main.py`
4. Создать Google таблицу → Поделиться → добавить `client_email` из JSON ключа с правами **редактора**
5. В первой строке таблицы вручную создать заголовок с названиями колонок (в том же порядке что в CSV)
6. В `config.py` заполнить:

```python
SHEETS_KEY_PATH = "service_account.json"
SHEETS_URL = "https://docs.google.com/spreadsheets/d/ВАШ_ID/..."
```

После настройки кнопки **Write KDA** и **Write All KDA** автоматически синхронизируют данные в облако. Если `SHEETS_URL` пустой — синхронизация молча пропускается, локальный CSV пишется в любом случае.

## API ключ

Публичный ключ уже вшит в `config.py`. Если перестанет работать — возьми актуальный из network-запросов на [lolesports.com](https://lolesports.com) (заголовок `x-api-key`).
